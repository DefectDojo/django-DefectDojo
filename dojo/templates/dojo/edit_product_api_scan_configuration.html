API Scan Configurations are supported for the test types SonarQube API, Cobalt.io API and Edgescan API.
  • For SonarQube API the field Service key 1 has to be set with the SonarQube project key. Service key 2 can be used for the Organization ID if using SonarCloud.
  • For Bugcrowd API the field Service key 1 has to be set with the Bugcrowd program code. Service key 2 can be set with the target in the Bugcrowd program (will be url encoded for the api call), if not supplied, will fetch all submissions in the program
  • For Cobalt.io API the field Service key 1 has to be set with the Cobalt.io asset id. Service key 2 will be populated with the asset name while saving the configuration.
  • For Edgescan API the field Service key 1 has to be set with the Edgescan asset id.
